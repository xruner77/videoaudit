# 代码审计技能 (Code Audit Skill)

当你完成代码修改并准备进入验证阶段时，应调用此技能。其目的是通过静态分析和逻辑推演，预判潜在的回退风险或新引入的缺陷。

## 审计准则

1. **副作用检查**：
    - 检查对公共组件（如 `Layout`, `App`）的修改是否会影响到意料之外的页面。
    - 确认全局状态（Context/Redux）或本地存储（LocalStorage）的读写逻辑是否一致。

2. **路由与生命周期**：
    - 确保组件在 `Keep-Alive` 模式下的 `isActive` 逻辑正确闭环。
    - 检查隐藏组件是否仍会执行昂贵的后台任务（除了必要的 Polling）。

3. **交互稳定性**：
    - 验证键盘监听器、事件委托等是否在组件隐藏时被正确禁用。
    - 检查弹出层（Modal/Portal）在跨标签页切换时的清理逻辑。

4. **异常边界**：
    - 思考如果网络请求在组件隐藏期间失败，重新激活时是否有合理的重试或报错机制。

## 执行步骤

1. 审查相关的 `diff` 记录。
2. 对照上述准则逐项排查受影响的组件。
3. 如果发现潜在隐患，立即在 `walkthrough.md` 中记录并在代码中修复。
4. 确保在最终提交 Git 前完成此审计过程。
