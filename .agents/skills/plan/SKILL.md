# 计划审核技能 (Plan Audit Skill)

当你需要对已制定的实施方案进行最终审核时，应调用此技能。其核心目标是确保方案的严谨性、安全性和向后兼容性。

## 核心原则

1. **零负面影响**：首要目标是确保新功能的引入不会破坏现有功能的正常运行。
2. **边缘情况挖掘**：主动寻找方案中未覆盖的特殊场景（如弱网、高频操作、并发冲突、权限越界等）。
3. **技术链审计**：检查从前端组件到后端接口，再到数据存储的全链路是否存在逻辑断裂。

## 审核清单

- [ ] **全局冲突**：检查是否引入了全局变量、全局监听器或可能导致样式冲突的 CSS。
- [ ] **生命周期安全**：对于长连接（如 WebSocket, Polling）或异步操作，确保在组件卸载或隐藏时有正确的清理逻辑。
- [ ] **权限检查**：验证方案是否破坏了现有的 RBAC（基于角色的访问控制）或数据加密策略。
- [ ] **性能损耗**：评估方案是否会导致明显的内存泄漏、CPU 占用激增或首屏加载变慢。
- [ ] **交互回退**：如果新功能依赖特定环境（如高性能 GPU、最新的浏览器 API），确保有合适的降级方案。

## 执行步骤

1. 阅读最新的 `implementation_plan.md`。
2. 针对上述“审核清单”进行逐一排查。
3. 如果发现问题，在方案中标记“审计发现”并提出修复策略。
4. 确保在 `task.md` 中包含必要的验证步骤，特别是针对现有功能的回归测试。
