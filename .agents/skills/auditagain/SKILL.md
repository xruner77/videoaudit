# 深度故障模拟审计 (AuditAgain Skill)

当你需要对复杂问题进行“刨根问底”式的深度审查时，或者常规审计无法解释异常现象时，调用此技能。

## 审计准则 (Execution Principles)

1. **多级回溯 (Multi-level Traceback)**：
    - 不仅检查当前变更，还要回溯该变量或状态的所有上游来源。
    - 验证数据在传递过程中的序列化/反序列化（如 Base64, JSON）是否存在隐含的精度丢失或截断。

2. **边界破坏测试 (Boundary Destruction)**：
    - 思考如果 3D 渲染频率突然降至 1fps（如系统节能模式），组件的状态同步是否会发生竞态或死锁。
    - 验证如果异步请求由于延迟极高而触发超时，内部状态机是否会卡在“加载中”。

3. **埋点式调试 (Instrumentation)**：
    - **主动埋点**：不要只在发现问题处加 Log。在状态流经的关键节点插入带时间戳的 `console.log` 或后端 `console.table`。
    - **性能监控**：在 `useFrame` 或高频处理函数中插入执行时长统计。

4. **因果链推演 (Causal Chain Inference)**：
    - 运用“5 Whys”法。如果图片消失，为什么？因为纹理被 dispose。为什么被 dispose？因为 useEffect 依赖变化。为什么依赖变化？...直到找到根源。

## 执行步骤 (Steps)

1. **静态扫描**：逐行分析受影响文件的业务逻辑。
2. **状态染色跟踪**：在纸上或注释中模拟一个状态变更从输入到输出的全过程。
3. **注入调试代码**：在可疑位置插入详细的调试日志。
4. **运行并复现**：在开发者控制台中观察详细的堆栈和时序信息。
5. **记录并加固**：在 `walkthrough.md` 中记录深度诊断的过程。
