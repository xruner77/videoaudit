# Git 提交技能 (Git Commit & Push Skill)

当你完成代码开发、审计和构建后，应调用此技能进行代码入库。其核心目标是确保提交信息清晰、中文显示正常且变更安全推送。

## 提交准则

1. **提交信息规范**：
    - 使用中文编写更新日志，描述变更的核心目标和影响范围。
    - 格式建议：`[模块名] 简短描述`
    - 示例：`[Frontend] 实现 Keep-Alive 状态持久化及切回自动刷新`

2. **防止中文乱码**：
    - 在运行 Git 命令前，确保终端字符集设置为 UTF-8（尤其是 Windows 环境）。
    - 统一使用双引号包裹提交消息：`git commit -m "中文内容"`。

3. **推送规范**：
    - 推送前先 `git pull` 同步远程代码（如适用）。
    - 推送完成后确认推送状态，确保远程仓库已更新。

## 执行步骤

1. `git add .` 将所有变更加入暂存区。
2. 运行 `git status` 检查待提交的文件列表是否正确。
3. 执行 `git commit -m "[中文提交信息]"`。
4. 执行 `git push` 将代码推送到远程仓库。
5. 记录提交记录，确保代码入库任务在 `task.md` 中被标记为完成。
