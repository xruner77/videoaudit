#### High-Risk Issues

**[HIGH] 缺少登录频率限制 -- 暴力破解风险**
- **Location:** `server/src/Routes/auth.php`, 第 19-57 行
- **Dimension:** Security & Robustness
- **Problem:** POST `/api/auth/login` 接口依然处于完全裸露状态且没有任何尝试限制。结合当前允许设置仅 6 位密码的系统策略下，攻击者只需使用自制脚本每秒发起数百次撞库登录请求，很容易试出后台管理员账户密码。
- **Fix:** 配置 Nginx 的 HTTP 限流区域对该接口进行防护拦截，或在 PHP 层面引入 Redis/限流组件。

**[HIGH] 文件上传过度信任客户端 MIME 类型声明**
- **Location:** `server/src/Routes/videos.php` (line 147), `comments.php` (line 182)
- **Dimension:** Security
- **Problem:** 校验中使用了 `$file->getClientMediaType()`，这只是客户端随请求附带的 Content-Type 请求头，极易被工具（如 BurpSuite）强行更改。虽然现行的代码采用了以扩展名（`mp4`, `jpg`等）结尾白名单和强制重命名的手段封堵了最致命的 PHP RCE 远程执行漏洞，但黑客依然可能把被恶意污染二进制病毒体或 HTML payload 包装在视频扩展名外壳中传上服务器惹事。
- **Fix:** 在服务器上使用 PHP 的原生 `finfo` 组件针对实际的上传临时物理文件做二进制验证。
```php
# Before: 
$clientMediaType = $file->getClientMediaType();
if (!in_array($clientMediaType, $allowedTypes)) { ... }

# After: (改用 finfo 读取服务器本地实际文件)
$finfo = new \finfo(FILEINFO_MIME_TYPE);
$realMime = $finfo->file($file->getFilePath());
if (!in_array($realMime, $allowedTypes)) { ... }
```

#### Maintainability Problems

**[MEDIUM] admin.vue 1,692 行上帝组件 (God Object)**
- **Location:** `src/pages/admin/admin.vue`
- **Dimension:** Architecture & Maintainability
- **Problem:** 该文件承载了管理员的仪表盘、视频列表、评论审查以及新加入的用户权限体系编辑等四个各自独立的功能。其代码跨幅已经突破一千六百多行大关，无论是数据响应、CSS 还是生命周期钩子都严重耦合在同一处。任何人接手新改动都有极大的心智负担。
- **Fix:** 按组件功能划分为四个独立的 Component (如 `AdminDashboard`, `AdminVideos` 等)，在父级仅做 Tab 切流与事件抛发。

**[MEDIUM] 零自动化测试覆盖**
- **Location:** 整个项目
- **Dimension:** Technical Debt Hotspots
- **Problem:** 代码库没有配备任何形式的单元测试或 e2e 验证框架。每次做完业务改版（例如之前的安全修正和缓存联调修正）全靠人工模拟环境点按去验证回归。
- **Fix:** 引入基础的测试框架，起码给认证模块和视频列表增减配上基础的冒烟断言测试。

**[MEDIUM] CORS 回退处理逻辑不规范**
- **Location:** `server/public/index.php`, 第 33 行
- **Dimension:** Robustness
- **Problem:** 如果 CORS 来源不在白名单之列，当前逻辑会退而求其次吐出 `($allowedOrigins[0] ?? '*')`。从安全结果上虽然对方浏览器依然会触发跨域拦截，但这种向不明来源坦白自己合法访问地址的手法不合常规，违反了正统的 CORS 处理模型。
- **Fix:** 当发现是未知来源时，直接丢弃且不返回 `Access-Control-Allow-Origin` 请求头。

#### Refactoring Priorities

1. **[P1 - High]** 追加登录限流器 —— addresses [HIGH #1] —— effort: S —— impact: 防止服务器因并发撞库攻击导致宕机和密码泄露。
2. **[P2 - High]** 切换 `finfo` MIME 检验 —— addresses [HIGH #2] —— effort: S —— impact: 彻底屏蔽多形态混淆的图片马/宏病毒文件驻留服务器硬盘。
3. **[P3 - Medium]** 剥离重构 Admin 子组件 —— addresses [MEDIUM #1] —— effort: M —— impact: 拆分成清晰的模块化文件流，降低未来二次迭代修改引起意外崩溃的可能。
4. **[P4 - Medium]** 拦截非法跨站探明 —— addresses [MEDIUM #3] —— effort: S —— impact: 让系统对外部嗅探保持全封闭式高冷状态，防止探测爬虫偷窥合法源地址。

**Quick Wins (fix in <1 hour):**
- CORS 阻截: 直接把跨源检查改为 `if (!in_array($origin, $allowedOrigins)) return $handler->handle($request);` 即可，只需耗时 2 分钟。
- 文件 MIME 检查: 用原生内置的 `finfo` 替换 `getClientMediaType` 的封装，几行判定逻辑互换就能解决。
